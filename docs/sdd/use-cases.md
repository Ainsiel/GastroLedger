# Use Cases

Each case records one representative successful, alternative and failed flow.

| ID | Use case | Primary actor | Success flow | Alternative flow | Failure flow |
|---|---|---|---|---|---|
| UC-001 | Register company tenant | Prospective administrator | **UC-001-S:** Valid company and admin data create an active pilot tenant and scoped session. | **UC-001-A:** An optional branch is created during onboarding. | **UC-001-F:** Duplicate company key or invalid credentials reject registration without partial data. |
| UC-002 | Configure tenant settings | Tenant administrator | **UC-002-S:** Base currency, locale and operational limits are updated and audited. | **UC-002-A:** Limits are reduced but existing branches remain active and new creation is blocked. | **UC-002-F:** Unsupported currency or unauthorized actor is rejected. |
| UC-003 | Manage users, invitations and roles | Tenant administrator | **UC-003-S:** A local user receives scoped roles and can access allowed branches. | **UC-003-A:** A time-limited invitation link is generated for manual sharing. | **UC-003-F:** Privilege escalation or expired invitation is rejected and audited. |
| UC-004 | Manage branches and warehouses | Tenant administrator | **UC-004-S:** A branch and its kitchen, bar or general warehouses are created. | **UC-004-A:** A warehouse is deactivated after confirming no open movements. | **UC-004-F:** Tenant branch limit or duplicate warehouse code blocks creation. |
| UC-005 | Manage units and conversions | Menu engineer | **UC-005-S:** Compatible purchase and consumption units receive a precise conversion factor. | **UC-005-A:** A new factor version takes effect on a future date. | **UC-005-F:** Zero, negative or dimension-incompatible conversion is rejected. |
| UC-006 | Manage ingredient catalog | Menu engineer | **UC-006-S:** Ingredient defines units, critical stock, shelf-life and costing policy. | **UC-006-A:** Ingredient is archived while historical records remain visible. | **UC-006-F:** Duplicate code or invalid unit mapping is rejected. |
| UC-007 | Manage suppliers and offers | Purchasing manager | **UC-007-S:** Supplier offer links ingredient, purchase unit, price and validity. | **UC-007-A:** A future offer is recorded without replacing the active offer early. | **UC-007-F:** Overlapping offer period or cross-tenant supplier is rejected. |
| UC-008 | Define sub-recipe | Chef | **UC-008-S:** Ingredients, quantities and expected yield create a versioned sub-recipe cost. | **UC-008-A:** A new version is scheduled while current production keeps the active version. | **UC-008-F:** Cycle, invalid yield or nesting beyond policy is rejected. |
| UC-009 | Define menu item and margin | Chef / analyst | **UC-009-S:** Menu recipe calculates theoretical cost, margin and suggested price. | **UC-009-A:** Branch-specific selling price changes margin without changing recipe. | **UC-009-F:** Missing active component cost prevents approval and explains gaps. |
| UC-010 | Recalculate theoretical costs | Background worker | **UC-010-S:** Accepted cost changes enqueue and publish affected recipe snapshots. | **UC-010-A:** Multiple changes are coalesced into one idempotent recalculation job. | **UC-010-F:** Failed job retains old snapshot, records failure and remains retryable. |
| UC-011 | Create production batch | Production lead | **UC-011-S:** FEFO input lots are consumed and a prepared sub-recipe lot is created with actual yield. | **UC-011-A:** Lower actual yield records production variance and reason. | **UC-011-F:** Insufficient input stock or inactive recipe prevents all movements. |
| UC-012 | Create purchase order | Purchasing manager | **UC-012-S:** Approved order is created from selected supplier offers. | **UC-012-A:** Reorder suggestions are edited before approval. | **UC-012-F:** Invalid quantities, ordering hold or unauthorized approval is rejected. |
| UC-013 | Receive supplier delivery | Warehouse operator | **UC-013-S:** Receipt creates accepted lots, costs and inventory movements against an order. | **UC-013-A:** Partial receipt keeps remaining order quantities open. | **UC-013-F:** Rejected temperature, duplicate lot or excess beyond tolerance blocks affected lines. |
| UC-014 | Record supplier return | Warehouse operator | **UC-014-S:** Return removes available quantity and records expected supplier adjustment. | **UC-014-A:** Partial return references only defective receipt lots. | **UC-014-F:** Quantity above available lot balance or closed supplier is rejected. |
| UC-015 | Request stock transfer | Branch manager | **UC-015-S:** Source and destination warehouses receive an approved transfer request. | **UC-015-A:** Approver reduces requested quantities before approval. | **UC-015-F:** Same source/destination, ordering hold or invalid item blocks request. |
| UC-016 | Dispatch stock transfer | Warehouse operator | **UC-016-S:** FEFO lots leave source availability and become in transit. | **UC-016-A:** Partial dispatch leaves remaining approved quantity pending. | **UC-016-F:** Insufficient stock or non-approved request creates no movement. |
| UC-017 | Receive stock transfer | Destination warehouse operator | **UC-017-S:** In-transit lots become destination stock with traceable lineage. | **UC-017-A:** Transport loss is recorded as variance with reason. | **UC-017-F:** Duplicate receipt or quantity above dispatched amount is rejected. |
| UC-018 | Record operational waste | Authorized operator | **UC-018-S:** Waste consumes a selected lot with reason, quantity and audit record. | **UC-018-A:** High-value waste enters approval before posting. | **UC-018-F:** Missing reason, unauthorized actor or excessive quantity is rejected. |
| UC-019 | Review expiry alerts | Warehouse operator | **UC-019-S:** Worker creates in-app FEFO alerts for lots nearing expiry. | **UC-019-A:** Operator acknowledges an alert and records an action note. | **UC-019-F:** Duplicate worker run creates no duplicate active alert. |
| UC-020 | Perform blind physical count | Branch manager / counter | **UC-020-S:** Blind counts are submitted, reviewed and approved variances post adjustments. | **UC-020-A:** Large variance requires recount before approval. | **UC-020-F:** Counter cannot view theoretical stock or approve own count. |
| UC-021 | Import or simulate sales consumption | Branch manager | **UC-021-S:** Valid sale lines explode active recipes and consume prepared/ingredient lots by FEFO. | **UC-021-A:** Shortage lines create allocation exceptions while valid lines commit. | **UC-021-F:** Duplicate import key, unknown menu item or invalid period rejects affected input. |
| UC-022 | Manage cash shift reconciliation | Branch manager / cashier | **UC-022-S:** Opening float and closing denomination count produce an operational variance. | **UC-022-A:** Manager records explanation for variance above threshold. | **UC-022-F:** Duplicate open shift or cashier closing another shift is rejected. |
| UC-023 | Plan shifts and record attendance | Workforce coordinator | **UC-023-S:** Weekly shifts and actual attendance produce worked-hour and absence report. | **UC-023-A:** Manager approves an overtime candidate or schedule exception. | **UC-023-F:** Overlapping assignment or unauthorized attendance correction is rejected. |
| UC-024 | Estimate royalties and control ordering holds | Franchise controller | **UC-024-S:** Recorded sales produce a non-financial royalty estimate by policy. | **UC-024-A:** Controller manually applies or releases an ordering hold with reason. | **UC-024-F:** Missing policy or unauthorized hold change is rejected. |
| UC-025 | Analyze cost variance and menu engineering | Analyst | **UC-025-S:** Period report compares theoretical consumption, actual movement and menu profitability. | **UC-025-A:** Analyst filters by branch, category or confidence status. | **UC-025-F:** Open imports or incomplete cost snapshots mark report incomplete instead of presenting false certainty. |
